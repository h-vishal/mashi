#Core Reference - MASHI_APP_PATH

The variable MASHI_APP_PATH inside the image paths (see following examples) represents the path to the 'application' directory of a Mashi app.

In case of an existing 'index.htm' in the 'application' directory (which is default) this path variable is empty (var MASHI_APP_PATH = '').
So MASHI_APP_PATH does not have to be set when used in 'index.html'.

If you want to place your app inside a sub-directory and embed in in more than one pages, set MASHI_APP_PATH like shown below indside the HEAD element of a document and always *before* embedding 'mashi.js'.

#####Example folder structure

	ROOT-DIR
	  index.php
	  sub-pages
	    tour.html
	  assets
	  my-mashi-apps
	    presentations
	      my-first-app
	        application
	        system


#####Example index.php:
	<html>
	  <head>
	    ...
	    <script type="text/javascript">
	      var MASHI_APP_PATH = 'my-mashi-apps/presentations/my-first-app';
	    </script>
		<script type="text/javascript" 
		  src="my-mashi-apps/presentations/my-first-app/system/javascripts/mashi.js">
		</script>
		...
	    
	
#####Example sub-pages/tour.html
	<html>
	  <head>
	    ...
	    <script type="text/javascript">
	      var MASHI_APP_PATH = '../my-mashi-apps/presentations/my-first-app';
	    </script>
		<script type="text/javascript" 
		  src="../my-mashi-apps/presentations/my-first-app/system/javascripts/mashi.js">
		</script>
	    ...
